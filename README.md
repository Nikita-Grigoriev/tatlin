# Магнитная лента и внешняя сортировка

Консольное приложение на C++, реализующее работу с лентами и алгоритм внешней сортировки данных с ограниченым объемом оперативной памяти.

## 🛠 Требования и сборка

Для сборки проекта необходим компилятор с поддержкой **C++20** и **CMake (>= 3.20)**.

⚠️ Для сборки тестового таргета требуется интернет.

```bash
mkdir build && cd build

cmake ..

cmake --build .
```

Будут сформированы исполняемые файлы:
* `apps/tatlin_app` — основное консольное приложение.
* `tests/tatlin_tests` — тесты.

## 💻 Использование

Запуск скомпилированного приложения из папки `build`:

```bash
./apps/tatlin_app <input_file> <output_file> <memory_limit> [config_path]
```

### Параметры:
* `<input_file>` — путь к исходному бинарному файлу.
* `<output_file>` — путь, по которому будет записан отсортированный результат.
* `<memory_limi>` — лимит оперативной памяти в байтах.
* `[config_path]` — опциональный путь к текстовому файлу конфигурации задержек.

### Пример файла конфигурации (`config.txt`):
Все значения задержек для эмуляции физических ограничений ленты указываются в миллисекундах:
```ini
read_delay = 5
write_delay = 5
shift_delay = 2
rewind_delay = 20
```

Запуск тестов из папки `build`:

```bash
./tests/tatlin_tests
```

## 🧪 Тестирование

Тесты:
* Построчный парсинг текстовой конфигурации задержек с валидацией некорректных значений.
* Граничные условия.
* Экстремальные лимиты памяти.
* Стресс-тесты
* Тесты задержек

Запуск модульных и интеграционных тестов:
```bash
cmake --build cmake-build-debug --target tatlin_tests
./cmake-build-debug/tests/tatlin_tests
```

## 🧠 Алгоритм
    Этап 1: Разбиение
        Лента input считывается порциями размера chunk_size (сколько влезает в RAM).
        Каждая порция сортируется в памяти.
        Полученные отсортированные куски циклически распределяются лентам первой группы временных лент.
    Этап 2: Слияние
        Элементы итеративно гоняются между первой и второй группами временных лент для эффективного слияния. Для слияния используется двочиная куча.
    Этап 3: Финальный перенос
        Итоговый отсортированный чанк оказывается на нулевой ленте одной из групп.
        Происходит копирование данных с временной ленты в output.
